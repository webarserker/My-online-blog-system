版本控制工具GIT

常用的git平台：

github，gitlab，gitea，gitee就是码云

git基础：

仓库代码操作的基本命令，README.md文件以及开源协议

git flow

分支管理，合并代码

GIT VS SVN

特点：

git 不用联网提交代码就能合并，断网解决本地冲突

svn 必须联网才能提交合并，否则无法提交

https VS ssh

特点：

https 每次下代码都需要账号+密码

ssh 不需要每次输入，因为电脑以及生成单独密匙，无需输入

ssh 新建本地密匙时，可以设置二级密码，别人就必须得到你的运行才能访问你的项目

Git基本操作：

git init：初始化并生成快照

git add . ：加入新文件 . 代表全部，也可以指定文件名称

执行这句命令后，git把这个文件移入git工作区

git commit -m “first commit”：合并提交

执行这句话才会给新文件拍摄快照，方便比对和提交，否则git默认不处理

git push：提交代码

基本操作，如果不执行git commit -m 则什么都不会提交

git pull: 更新代码，从线上更新，冲突会提示

git log: 查看是谁提交了代码，主要看hash

git reflog: 查看了删除了哪些内容，主要看hash

核心命令：可以查看本地所有提交

git reset –hard 哈希值：即可回退版本

在git log里面项目日志就回退到这个版本前了，但是reflog里面记录依旧

git config –global –list： 查看当前用户

git config –global user.name “XXX”

git config –global user.email “xxx@.com”

以上可以指定用户名去提交代码

git status：查看当前分支状态

git remote -v：查看当前仓库

git stash: 保存到缓冲区的某个区，本地修改代码，但是不想提交时可以吧修复文件置为此状态

git stash apply：找回缓存区代码，代码写完了，又想提交，可以吧状态改回来

git remote add 自定义名称 git@github.com:webarserker/test2.git：指定提交代码的仓库地址

再执行git push 自定义名称，即可推送到多个代码仓库

git diff: 看哪些代码更新过，有UI版本，但是都是基于这个命令执行的

git Flow 无用论：https://blog.csdn.net/millionmanhjq/article/details/106121984

假设我们公司就我一个前端开发工程师，不是多人开发，所以我不需要flow。有几个前端，但是不是多人协同啊，还是我一个人写代码，所以还是不用这个功能

经典模型问题：

必须使用dev分支：假如你要改master分支，你必须先创建dev分支，然后修改提交到dev，最后master和dev合并

复杂度高：hotfix与release分支
