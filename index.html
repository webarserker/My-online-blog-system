Docker容器技术的特点：文件系统隔离：每个进程容器运行在完全独立的根文件系统里。
资源隔离：可以使用cgroup为每个进程容器分配不同的系统资源，
例如CPU和内网络隔离：每个进程容器运行在自己的网络命名空间里，拥有自己的虚拟接口和写时复制：
采用写时复制方式创建根文件系统，这让部署变得极其快捷，并且节省硬盘空间。
日志记录：Docker将会收集和记录每个进程容器的标准流
（stdout/stderr/stdin于实时检索或批量检索。变更管理：容器文件系统的变更可以提交到新的映像中，
并可重复使用以创建更多器。无需使用模板或手动配置。交互式Shell：Docker可以分配一个虚拟终端并关联到任何容器的标准输入上，
例一个一次性交互shell。
所以：ABC对，而D，Docker采用的是资源共享型的管理方案，所有的Docker使件资源由docker daemon进行管理与分配。

官方源拉裤里了：sudo curl -L “https://github.com/docker/compose/releases/download/1.26.2/docker-compose-$(uname -s)-$(uname -m)” -o /usr/local/bin/docker-compose

免除权限：sudo chmod +x /usr/local/bin/docker-compose

docker：一定要先写 systemctl start docker 启动docker，然后设置systemctl enable docker 开机自启动

然后安装：docker pull mongo：4 这个版本一般是 https://hub.docker.com/_/mongo 带latest的版本号 但是安装不了

改成国内的加速源：vim /etc/docker/daemon.json 加上这句话 阿里的最好用
{
“registry-mirrors”: [“https://9cpn8tt6.mirror.aliyuncs.com"]
}

再执行：
systemctl daemon-reload
systemctl restart docker

接下啦要搞定防火墙：
方案一：关掉防火墙，放行所有的端口，这样不安全
方案二：开发某个端口，这样很安全。采取方案二

本地好像是不需要弄防火墙就能接受到服务

执行：firewall-cmd –zone=public –add-port=10050/tcp –permanent
success

查看防火墙的通道端口：firewall-cmd –list-all

执行：firewall-cmd –reload
success

执行：firewall-cmd –state
running
